Smack That Rooster

Game Programmer — Multiplayer Combat Game — Health, Match Flow, Victory Conditions and Game Feel

Gameplay

Game Overview

Smack That Rooster is a chaotic multiplayer combat game focused on quick matches, direct confrontation and arcade-style game feel. Players fight each other in fast-paced rounds where health management, timing and match flow define the experience.

My Role — Game Programmer

I worked mainly as a Game Programmer, focusing on player health, match timer logic, victory conditions, game data handling and the end-of-match flow. I also contributed to UI and visual presentation elements related to match results.

Gameplay Systems Programming

  • Designed and implemented the player health system.
  • Built the match timer and round flow logic.
  • Implemented victory conditions and end-of-match behavior.

Game Data, UI & Feedback

  • Handled player and match state data during gameplay.
  • Created logic for the end-of-game screen and result flow.
  • Contributed to UI and visual elements related to match results.

Core Gameplay

Smack That Rooster is built around short, chaotic multiplayer fights. Players enter a match, attack each other, manage their health, survive the timer and compete to trigger the victory condition.

Gameplay Pillars

Chaotic Multiplayer Fast Matches Arcade Combat Clear Win Conditions Readable UI Feedback Strong Game Feel

Core Loop

Enter the arena, fight other players, reduce their health, survive the match timer and secure victory through the end-game condition.
